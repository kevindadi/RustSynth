## Installation

### From Source

```bash
# Clone the repository
git clone https://github.com/example/RustSynth.git
cd RustSynth

# Build release version
cargo build --release

# Optional: Add to PATH
export PATH="$PATH:$(pwd)/target/release"
```

### Using Docker

```bash
docker build -t RustSynth .
docker run --rm -v $(pwd)/examples:/workspace RustSynth
```

### Prerequisites

- **Rust stable** (1.85+): For building RustSynth
- **Rust nightly**: For generating rustdoc JSON (required for `--output-format json`)
- **Graphviz** (optional): For visualizing DOT files

```bash
# Install Rust nightly
rustup toolchain install nightly

# Install Graphviz (macOS)
brew install graphviz

# Install Graphviz (Ubuntu/Debian)
sudo apt install graphviz
```

## Preparing Your Crate

### Step 1: Create a Library Crate

RustSynth works with library crates. Create a simple example:

```rust
// src/lib.rs
pub struct Counter {
    value: i32,
}

pub const fn make_counter() -> Counter {
    Counter { value: 0 }
}

impl Counter {
    pub const fn new() -> Self {
        Counter { value: 0 }
    }

    pub fn inc(&mut self) {
        self.value += 1;
    }

    pub fn get(&self) -> i32 {
        self.value
    }
}
```

### Step 2: Generate rustdoc JSON

```bash
cargo +nightly rustdoc -Z unstable-options --output-format json --lib
```

This creates `target/doc/<crate_name>.json`.

### Step 3: Verify JSON Output

```bash
# Check the file exists
ls -la target/doc/*.json

# Optionally inspect the structure
head -c 1000 target/doc/my_crate.json | jq .
```

## Task Configuration

Create a `task.toml` file to specify synthesis parameters:

### Minimal Configuration

```toml
[inputs]
doc_json = "target/doc/my_crate.json"

[search]
max_steps = 50

[goal]
want = "own i32"
count = 1
```

### Full Configuration

```toml
[inputs]
doc_json = "target/doc/my_crate.json"

[search]
stack_depth = 8           # Max borrow stack depth (default: 8)
default_place_bound = 2   # Max tokens per place (default: 2)
max_steps = 100           # Max simulation steps (default: 100)

# Override bounds for specific places
[search.place_bounds]
"own_i32" = 5             # Allow more i32 values
"own_Counter" = 2         # Limit Counter instances

[filter]
# Allowlist: Only use these functions (leave empty to allow all)
allow = [
    "make_counter",
    "Counter::new",
    "Counter::inc",
    "Counter::get",
]

# Denylist: Never use these functions
deny = [
    "internal_helper",
    "unsafe_operation",
]

[goal]
want = "own i32"          # Target: capability + type
count = 1                 # How many to produce

[initial]
# Pre-populate places with tokens (optional)
tokens = [
    { type = "i32", count = 1 }
]
```

### Goal Specification

The `want` field specifies what you want to synthesize:

| Goal | Description |
|------|-------------|
| `own i32` | An owned i32 value |
| `own Counter` | An owned Counter instance |
| `own &Counter` | An owned shared reference |
| `own &mut Counter` | An owned mutable reference |
| `frz Counter` | A frozen (borrowed) Counter |

## Running the Synthesizer

### Basic Synthesis

```bash
RustSynth synth \
    --doc-json target/doc/my_crate.json \
    --task task.toml \
    --out output.rs
```

### Step-by-Step Pipeline

```bash
# 1. Generate API Graph
RustSynth apigraph -i target/doc/my_crate.json -o output/

# 2. Generate PCPN model
RustSynth pcpn -i target/doc/my_crate.json -o output/

# 3. Run simulation
RustSynth simulate -i target/doc/my_crate.json --max-steps 50

# 4. Generate code
RustSynth generate -i target/doc/my_crate.json -o output/
```

### Reachability Analysis

```bash
# Generate reachability graph
RustSynth reachability \
    -i target/doc/my_crate.json \
    -o output/ \
    --max-states 100 \
    --max-stack 5

# Visualize with Graphviz
dot -Tpng output/reachability.dot -o output/reachability.png
```

## Understanding Output

### Generated Code Structure

```rust
//! Generated by RustSynth PCPN Synthesizer

#![allow(unused_variables)]
#![allow(unused_mut)]
#![allow(dead_code)]

fn main() {
    let counter_0 = make_counter();           // Create value
    let mut_counter_1 = &mut counter_0;       // Borrow mutably
    counter_0.inc();                          // Call method
    drop(mut_counter_1);                      // End borrow
    let i32_2 = counter_0.get();              // Get result
}
```

### Trace Output

```
=== Trace (5 steps) ===
  1. make_counter [+v0]           # Created token v0
  2. borrow_mut(Counter) [-v0] [+v1]  # Consumed v0, produced v1
  3. Counter::inc [-v1]           # Used mutable reference
  4. end_mut(Counter) [-v1] [+v0] # Restored ownership
  5. Counter::get [-v0] [+v2]     # Got i32 result
```

### DOT Files

- **apigraph.dot**: Bipartite graph of functions and types
- **pcpn.dot**: Colored Petri Net with places and transitions
- **reachability.dot**: State space exploration graph

## Advanced Usage

### Filtering Functions

```toml
[filter]
# Only synthesize using these specific functions
allow = ["safe_api::*"]

# Exclude dangerous operations
deny = ["*::unsafe_*", "*::internal_*"]
```

### Custom Place Bounds

```toml
[search.place_bounds]
# Limit expensive types
"own_BigStruct" = 1

# Allow more primitives
"own_i32" = 10
"own_bool" = 5
```

### Module Filtering (CLI)

```bash
# Only analyze specific modules
RustSynth synth \
    --doc-json crate.json \
    --task task.toml \
    --module my_module \
    --module other_module
```

### Batch Processing

```python
#!/usr/bin/env python3
import subprocess
import glob

for json_file in glob.glob("examples/*/target/doc/*.json"):
    subprocess.run([
        "RustSynth", "generate",
        "-i", json_file,
        "-o", f"output/{json_file.split('/')[1]}"
    ])
```

## Troubleshooting

### Common Issues

#### "No witness found"

- **Cause**: Search space exhausted without reaching goal
- **Solutions**:
  - Increase `max_steps` in task config
  - Increase `stack_depth` for complex borrow patterns
  - Reduce constraints in `[filter]`
  - Check if goal is achievable with available API

#### "Failed to load rustdoc JSON"

- **Cause**: JSON format incompatible or file not found
- **Solutions**:
  - Ensure using nightly: `cargo +nightly rustdoc ...`
  - Check file path is correct
  - Verify rustdoc-types version compatibility

#### "Type not found"

- **Cause**: Type not exported publicly
- **Solutions**:
  - Make type `pub` in source crate
  - Check module visibility

### Debug Mode

```bash
# Enable debug logging
RUST_LOG=debug RustSynth synth --doc-json ... --task ...

# Trace-level logging
RUST_LOG=trace RustSynth synth --doc-json ... --task ...
```

### Performance Tips

1. **Reduce place bounds** for large state spaces
2. **Use filters** to exclude irrelevant APIs
3. **Start with small `max_steps`** and increase gradually
4. **Profile with `--max-states`** in reachability mode

## API Reference

### CLI Options

```
RustSynth <COMMAND>

Commands:
  synth        Run synthesis with task configuration
  apigraph     Generate API Graph
  pcpn         Generate PCPN model
  simulate     Run simulator
  reachability Generate reachability graph
  generate     Full pipeline
  all          Generate API Graph and PCPN

Global Options:
  -h, --help     Print help
  -V, --version  Print version
```

### Environment Variables

| Variable | Description |
|----------|-------------|
| `RUST_LOG` | Logging level (error, warn, info, debug, trace) |
| `RustSynth_CONFIG` | Default task config path |
